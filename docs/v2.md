# Quick Send v2

## Phase 1

- Keep the existing Quick Send room/message model.
- Run Quick Send inside a Tauri shell on macOS.
- Add stable native device identity through `/api/identity/ensure`.
- Keep browser fingerprint identity working for the current web app.
- Use the migrated AltSendme/Iroh engine for native file sharing.
- Keep browser file upload/download on the existing HTTP cache path.

## Identity

Web clients keep sending `visitorId`.

Native apps should send `deviceId`. The server stores only `device_id_hash`, so
the raw device id never lands in SQLite.

```json
{
  "deviceId": "native-stable-device-id"
}
```

## Transport

`engine/` and `src-tauri/` come from `tonyantony300/alt-sendme`.

Native file sharing uses the Tauri `send_items` command to create an Iroh
ticket, then posts the ticket into the current chat room as a text message.

Skipped for phase 1: typed file-ticket messages, receiving tickets in-app,
multi-room sync over Iroh gossip, and iOS packaging. Add those after macOS dev
can open the chat UI and share one Iroh ticket from a room.
